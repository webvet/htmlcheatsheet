Cheatsheet - Using CSS

An insiders guide to using stylesheets

What is CSS?

CSS is the acronym for Cascading Style Sheets.

It is used to describe the presentation of a document written in a markup language, such as HTML.

It was developed to allow the separation of presentation (including layout, colours and fonts) and the content of webpages.


Why use CSS?

Explain the practice of separating html, css and js in different files. One change can then affect multiple pages.


How to use CSS

The three methods of using - inline, in head and external - are outlined in the table below:

Table outlining the three methods of using CSS in webpages
Method Example Uses
Inline CSS <p style="color: red";>Text here</p> Might be used in a very simple, single webpage with only little styling requirements. If a website styled this way were to expand and develop, it makes changing colours, fonts, sizes etc more difficult and time-consuming.
Embedded CSS Within <style> tags, often placed in the <head> section Blocks of CSS can be written in this fashion, and is kept seperated from the HTML content in the <body> of the code. However, needs to be placed within every> new webpage inidividually, and therefore if any change is made, it has to be written to every page.
External CSS <link href="style.css" rel="stylesheet"> links to an external file where changes made will apply to all relevant webpages. By separating the HTML and CSS code completely, this is generally the recommended method. It allows a single change in the CSS file to affect every page it is used in.

Final Thoughts

This, hopefully, briefly outlines what CSS is and how to use it. As a final thought, it is worth being aware that there is a heirarchy of how CSS is interpretted.

Inline CSS is loaded fastest, as the browser does not require to load any other informaiton (eg from an external file). And it takes priority of embedded and externally declared styles.

Next in line are any embedded styles, and finally the external sheet is read. The result is that an element could have a style set using all three - but the inline style will be used and rendered by the browser.
